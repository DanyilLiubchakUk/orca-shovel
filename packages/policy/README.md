# packages/policy

Policy, tools, capabilities, and approvals.

Responsibilities:

- tool registry
- capability registry
- risk levels
- approval requirements
- frequency and quiet-hour rules
- admin review request shaping
- action plan validation
- settings registry and chat-powered management policy

Rule: AI can propose actions, but this package decides what is allowed to execute.

Chat is another UI for typed tools. It cannot bypass validation, limits, confirmation, or audit logging.
